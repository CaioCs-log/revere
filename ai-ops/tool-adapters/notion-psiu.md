# Adapter — Psiu / Notion

## Papel recomendado

Psiu e Notion devem operar como cockpit executivo:

- classificar pedido;
- identificar projeto e modo;
- escolher pacote minimo;
- consultar matriz AI-OPS;
- preparar contrato para executor;
- registrar fechamento;
- manter memoria operacional.

## Quando consultar contexto amplo

Somente em:

- auditoria;
- reorganizacao estrutural;
- diagnostico;
- mudanca de instrucoes;
- conflito entre fontes.

Em tarefas normais, usar pacote minimo.

## Regra de parada

Se o pacote minimo nao bastar, Psiu deve declarar:

```plain text
documento adicional necessario -> motivo -> pergunta objetiva que ele responde
```

## Nao fazer

- Nao editar repo diretamente.
- Nao marcar tarefa como concluida sem evidencia.
- Nao criar banco paralelo de tarefas.
- Nao transformar preparacao/teste em execucao sem autorizacao.
